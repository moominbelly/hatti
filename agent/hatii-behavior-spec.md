# 하띠 (Hatti) — 앱 동작 정의서

> 앱이 **무엇을 보여주고, 무엇에 반응하고, 언제 무엇이 바뀌는지**를 화면·상태 단위로 정의한 문서.
> 기준: React 프로토타입에서 검증된 흐름 + Supabase/Gemini 설계(`hatti_dev_spec.md`)
> 용도: 개발 착수 기준 / 발표 설명 기준

---

## 1. 한눈에 보는 전체 흐름

```
    [앱 실행]
        │
        ├─ 최초 실행? ─── Yes ──▶ 익명 로그인 (자동, 화면 없음)
        │                              │
        ▼◀─────────────────────────────┘
    ┌───────────┐
    │ ① 홈      │  하띠 · 친밀도 · 스트릭 · 시간대 · 최근 감정
    └─────┬─────┘
          │ [체크인 시작하기]
          ▼
    ┌───────────┐
    │ ② 입력    │  시간대별 질문 + 자연어 입력
    └─────┬─────┘
          │ [하띠에게 들려주기]
          ▼
    ┌───────────┐
    │ ③ 분석중  │  하띠가 마음을 읽는 중 (2~4초)
    └─────┬─────┘
          │
          ├──── 위기 감지 ────▶ ┌───────────┐
          │                     │ ⑤ 위기    │ 전문 리소스 연결
          │                     └─────┬─────┘
          │                           │ [홈으로]
          ▼                           │
    ┌───────────┐                     │
    │ ④ 응답    │  감정 → 공감 → 확언  │
    └─────┬─────┘                     │
          │ [체크인 마치기]            │
          ▼◀───────────────────────────┘
    ┌───────────┐
    │ ① 홈      │  친밀도 +1 · 스트릭 갱신 · (성장/마일스톤 시 토스트)
    └───────────┘
```

**핵심:** 정상 경로는 ①→②→③→④→① 의 단일 루프. 위기 경로만 ⑤로 빠지며, **이때 친밀도·스트릭은 오르지 않는다.**

## 2. 화면별 동작 정의

### ① 홈

| 구분 | 내용 |
|------|------|
| **표시 데이터** | 하띠 캐릭터(성장 단계별), 단계명(`Lv.2 · 아기 하띠`), 친밀도(💛 n), 스트릭(🔥 n일), 시간대(`🌙 저녁 · 21:34`), **오늘의 날씨 칩**, 하띠 인사말, 최근 감정 칩 최대 4개, **오늘의 카드 슬롯**(체크인 후) |
| **액션** | `[체크인 시작하기]` → ② 입력 / **날씨 칩 탭 → 날씨 선택** / **하띠 탭 → 쓰다듬기** / **카드 슬롯 탭 → 오늘의 카드 뽑기** |
| **데이터 출처** | Supabase `hatti_state` + `checkin_log` 직접 조회 (RLS로 본인 것만) |

> **설계 원칙:** 선택 요소(날씨·쓰다듬기·카드)는 **전부 홈에 둔다.** 체크인 플로우 안에
> 넣고 "건너뛰기"를 주면 유저는 여전히 그만큼의 단계로 체감한다(스킵도 결정이다).
> **코어 3단계(일기 → 분석 → 공감·확언)는 늘리지 않는다.**

**상태별 UI**

| 상태 | 동작 |
|------|------|
| 정상 | 위 데이터 전부 표시 |
| 로딩 | 하띠는 즉시 표시, 수치·히스토리는 스켈레톤 (캐릭터가 먼저 보여야 앱이 살아있다고 느낌) |
| 빈 데이터 (첫 유저) | 친밀도 0, 스트릭 0, 최근 감정 영역 **숨김**. 인사말은 첫 만남 버전 |
| 에러 | 마지막 캐시값 표시 + 조용한 재시도. 하띠 화면에 에러 다이얼로그를 띄우지 않는다 |
| 오프라인 | 캐시값으로 렌더. 체크인 버튼 누르면 안내 |

> **원칙:** 홈은 **절대 에러 화면이 되지 않는다.** 하띠를 만나러 왔는데 에러가 반기면 안 됨.

---

### ② 입력

| 구분 | 내용 |
|------|------|
| **표시 데이터** | 하띠(축소), 시간대별 질문 1개(랜덤), 입력창 |
| **액션** | 텍스트 입력 / `[하띠에게 들려주기]` → ③ / `[← 돌아가기]` → ① |
| **입력 검증** | 공백만 있으면 버튼 비활성 / **최대 500자** (초과 입력 자체가 막힘) |

**글자수 카운터 노출 규칙**

- 400자를 넘어설 때부터 `n자 남았어`가 조용히 나타난다. 한계 도달 시 `여기까지 담을 수 있어`.
- **처음부터 카운터를 보여주지 않는다.** 글자수가 계속 보이면 "잘 써야 한다"는 압박이 생기고,
  그건 이 앱이 없애려는 부담(일기 앱이 3일 만에 버려지는 이유) 그 자체다.
- 500자 제한은 **서버에서도 검증**한다. 클라이언트 제한은 우회될 수 있고, 긴 입력은
  토큰 비용·지연을 늘리며 프롬프트 인젝션 표면을 키운다.

**시간대별 질문**

| 시간대 | 성격 | 예시 |
|--------|------|------|
| 아침 (05:00~11:59) | 의도 설정 | "오늘, 어떤 마음으로 하루를 시작하고 싶어?" |
| 저녁 (그 외) | 감정 회고 | "오늘 하루, 어땠어? 가장 마음에 남는 순간을 들려줘." |

- 질문은 각 세트에서 **랜덤 선택** — 매일 같은 질문이면 3일 만에 질린다.
- 플레이스홀더: `"떠오르는 대로 편하게 적어줘. 문장이 아니어도 괜찮아."` (라벨링 부담 제거가 제품 가치)

### ③ 분석중

| 구분 | 내용 |
|------|------|
| **표시** | 하띠 + `"하띠가 네 마음을 읽는 중…"` + 점 3개 애니메이션 |
| **지속** | Gemini 1차+2차 호출 시간 (2~4초 예상) |
| **액션** | 없음 (뒤로가기 차단 — 중복 호출 방지) |

**상태별 동작**

| 상황 | 동작 |
|------|------|
| 성공 | → ④ 응답 |
| 위기 감지 | → ⑤ 위기 |
| 타임아웃(10초 초과) | "하띠가 잠깐 딴생각을 했나 봐" + 재시도 버튼 |
| 네트워크 실패 | 동일. **입력 텍스트는 보존** — 다시 쓰게 하면 안 됨 |
| 쿼터 초과(429) | 동일 문구로 처리. 유저에게 "API 한도"를 노출하지 않는다 |

> **원칙:** 실패해도 하띠의 목소리로 말한다. 기술 용어(에러코드, API, JSON)를 유저에게 보여주지 않는다.

### ④ 응답 (핵심 화면)

**표시 순서 — 이 순서가 "Name it to tame it"의 재현이다**

```
1. 감정 라벨 + 강도        ← Name  (감정에 이름이 붙는 순간)
2. 맥락 태그 (#회의 과부하)
3. 하띠 공감 대사           ← Tame  (혼자가 아님)
4. [확언 카드 받기] 버튼
5. 확언 손편지 카드         ← Reframe (인지 전환)
```

| 구분 | 내용 |
|------|------|
| **표시 데이터** | 감정 라벨(피로/불안/분노/슬픔/기쁨/잔잔함), 강도 5점 도트, 맥락 태그, 하띠 대사(2문장 이내), 확언 |
| **액션** | `[확언 카드 받기]` → 카드 공개 / `[체크인 마치기]` → ① 홈 |
| **연출** | 하띠 표정·볼터치가 감정 색으로 변화. 확언 카드는 손글씨체 + 등장 애니메이션 |

**확언 카드를 버튼으로 분리한 이유:** 한 번에 다 쏟으면 확언이 묻힌다. 유저가 직접 열게 해서 **받는 경험**으로 만든다.

**체크인 마치기 시 발생하는 것**
1. 친밀도 +1 (누적, 감소 없음)
2. 스트릭 갱신 (아래 3장 규칙)
3. 홈으로 이동
4. 조건 충족 시 토스트:
   - 성장: `🌱 하띠가 자랐어! 이제 «아기 하띠»`
   - 마일스톤: `🎉 7일 연속! 하띠가 특별한 인사를 준비했어`

### ⑤ 위기 (안전 분기)

| 구분 | 내용 |
|------|------|
| **진입 조건** | 키워드 프리필터 **또는** AI `crisis_flag` **또는** AI 안전필터 차단 (3중 방어, OR 결합) |
| **표시** | 하띠(슬픈 표정) + `"잠깐, 지금 마음이 많이 힘든 것 같아."` + 전문 리소스 |
| **리소스** | 자살예방상담전화 **109** (24시간) / 정신건강상담 1577-0199 / 청소년 1388 |
| **액션** | `[홈으로 돌아가기]` |

**이 화면에 없는 것 (의도적)**
- ❌ 공감 대사 — "공감 only" 원칙의 **유일한 예외**
- ❌ 확언 카드
- ❌ 친밀도·스트릭 증가 — 안전 상황에 게임화를 섞지 않는다
- ❌ 하띠의 "기억" 소환

> **설계 근거:** 자해 신호에 하띠가 공감으로 답하면 안 된다. **사람의 도움으로 연결**하는 것이 유일한 정답. 하띠는 여기서 물러난다.

### ⑥ 마음 기록 (히스토리)

| 구분 | 내용 |
|------|------|
| **진입** | 홈의 "최근 마음 기록 ›" 영역 탭 |
| **표시 데이터** | 날짜별로 묶인 체크인 기록 — 감정 이모지+라벨+강도 도트, 맥락 태그, 날씨, 하띠 대사, 확언 |
| **정렬** | 최신순. 날짜가 바뀌는 지점에 `오늘`/`어제`/`7월 15일` 헤더 |
| **액션** | `[← 돌아가기]` → ① 홈 |
| **빈 상태** | 하띠 + `"첫 마음을 들려주면 여기에 차곡차곡 쌓일 거야."` |

**이 화면에 없는 것 (의도적)**
- ❌ 그래프·차트·평균·패턴 분석 — **하띠의 성격은 분석이 아니라 공감이다.**
  통계 대시보드는 무드 트래커의 문법이고, 그걸 넣으면 Daylio와 같아진다.
- ❌ 기록 수정·삭제 (MVP 범위 밖)
- 위기 기록은 목록에 노출하지 않는다 (`crisis_flag = false`만 조회)

> 이 화면의 목적은 "데이터 확인"이 아니라 **지나온 마음을 조용히 다시 보는 것**이다.

## 3. 감정 캐릭터

감정 9종에는 각각 **직접 그린 얼굴**(`EmotionFace` 위젯)과 고유 색상이 있다.
하띠 본체와 같은 CustomPainter 방식이라 아트 스타일이 통일되고, 크기를 키워도 깨지지 않는다.

| 감정 | 색상 | 표정 설계 | 액센트 |
|------|------|-----------|--------|
| 기쁨 joy | 골드 | 활짝 웃는 아치 눈 + 큰 미소 | 볼터치 |
| 편안 calm | 아쿠아민트 | 부드럽게 감은 눈 + 작은 미소 | 볼터치 |
| 뿌듯 pride | 더스티로즈 | 웃는 눈 + 자신감 있는 미소 | **반짝임 ✦** |
| 피로 fatigue | 앰버브라운 | 반쯤 감긴 처진 눈 | **한숨 입** |
| 불안 anxiety | 라벤더 | 안쪽이 올라간 걱정 눈썹 + 물결 입 | **땀방울** |
| 분노 anger | 코랄레드 | 찌푸린 눈썹 + 굳은 일자 입 | — |
| 슬픔 sadness | 블루 | 처진 눈 + 아래로 굽은 입 | **눈물** |
| 자책 guilt | 토프 | 아래를 보는 시선 + 다문 입 | — |
| 잔잔함 neutral | 세이지 | 점 눈 + 짧은 일자 입 | — |

**설계 원칙**
- 작은 크기(20~26px)에서도 구분되어야 하므로, 눈·입만으로 부족한 감정에는 **고유 액센트**를 준다.
- 유니코드 이모지는 UI에 쓰지 않는다. (기기·OS마다 모양이 달라 브랜드 일관성이 깨짐)
  `Emotion.emoji`는 위젯을 렌더할 수 없는 **텍스트 전용 맥락**(푸시 알림 등)의 폴백일 뿐이다.
- 사용처: 홈 감정 칩(20px), 히스토리 타일(26px), 응답 화면 감정 칩(28px)
- 하띠 캐릭터 본체의 표정은 별개다(4종 그룹핑: 활짝웃음/잔잔한미소/시무룩/굳은일자입).

## 4. 캐릭터 상태 규칙

### 친밀도 (Intimacy)
- 체크인 1회 = **+1**
- **감소하지 않는다.** 며칠 안 와도 그대로.
- 근거: 방치 패널티는 불안을 유발 → 심리앱에서 모순 (설계 원칙 "보상 only")

### 성장 단계 (Stage)

| 단계 | 조건 | 이름 | 비주얼 |
|------|------|------|--------|
| 1 | 친밀도 0~2 | 새싹 하띠 | 작음, 새싹 1개 |
| 2 | 친밀도 3~6 | 아기 하띠 | 중간, 새싹 2개 |
| 3 | 친밀도 7+ | 하띠 | 큼, 새싹 2개 + 꽃 |

- 말투도 단계별로 변화 (친밀도가 프롬프트에 주입됨)

### 스트릭 (Streak)

| 상황 | 결과 |
|------|------|
| 첫 체크인 | 1 |
| 어제 체크인함 | +1 |
| **오늘 이미 체크인함** | **유지** (하루 2회 해도 중복 증가 없음) |
| 하루 이상 공백 | **1로 리셋** |

- **날짜 기준: KST(UTC+9).** 서버는 UTC로 도므로 밤 체크인이 잘못 끊길 수 있음.
- 마일스톤: **3일 / 7일 / 14일** 도달 시 보상 해금(특별 인사, 배경색, 표정 세트)

### 하띠의 "기억" (SHOULD)
- 같은 감정을 최근에 느낀 기록이 있으면 하띠 대사에 스치듯 반영
- 발동 조건: 친밀도 3 이상 · 1~30일 전 · neutral 제외 · 위기 아님
- 예: `"지난번에도 회의가 너를 참 많이 데려갔었지"`
- ❌ 금지: `"또 회의 때문에 지쳤네"` (기억이 아니라 평가)

## 4. 시간대 자동 판정

| 시각 | 시간대 | 질문 성격 |
|------|--------|-----------|
| 05:00 ~ 11:59 | 아침 | 의도 설정 |
| 12:00 ~ 04:59 | 저녁 | 감정 회고 |

- **유저가 고르지 않는다.** 접속 시각으로 자동 결정 (설계 원칙)
- 홈 상단에 감지 결과만 읽기전용 표시
- 앱을 켜둔 채 경계 시각을 넘으면 자동 전환
- 하루 2회(아침+저녁) 체크인 유도가 목적이지만, **강제하지 않는다**

## 4-1. 선택 요소 (홈 상시 인터랙션)

셋 다 **하지 않아도 아무 불이익이 없다.** 스트릭·친밀도는 **코어 체크인 완료에만** 연동된다.
(선택 요소를 해야 스트릭이 오르면 그건 선택이 아니라 필수다)

### 오늘의 날씨 (수동 선택)

| 구분 | 내용 |
|------|------|
| 위치 | 홈 상단 칩 |
| 동작 | 탭 → 6종 중 선택 (맑음/구름/비/눈/바람/안개) |
| 권한 | **불필요** (위치권한 없음) |
| 용도 | 체크인 시 하띠 공감 대사의 **컨텍스트로 주입** |
| 미선택 시 | 칩에 `날씨?` 표시. 프롬프트에 아무것도 넣지 않음 |

- 하루 단위로 저장. 다음 날 초기화.
- 하띠 대사 예: *"비 오는 날이라 마음이 더 가라앉았나 보다."*
- ⚠️ 날씨를 **감정의 원인으로 단정하지 않는다.** 어디까지나 곁들이는 배경.

### 하띠 쓰다듬기

| 구분 | 내용 |
|------|------|
| 위치 | 홈의 하띠 캐릭터 |
| 동작 | 탭 → 하띠가 반응(살짝 흔들림 + 말풍선 변화) |
| 보상 | **없음(수치 증가 없음)** — 반응 자체가 보상 |
| 제한 | 없음. 몇 번이든 가능 |

> **왜 친밀도를 주지 않나:** 친밀도는 "마음을 들여다본 횟수"의 지표다. 탭으로 올릴 수 있으면
> 의미가 희석되고, 유저가 수치를 위해 연타하는 그라인딩이 된다. 쓰다듬기는 **정서적 교감**
> 그 자체로 완결시킨다. (다마고치의 본질)

### 오늘의 카드 (타로풍)

| 구분 | 내용 |
|------|------|
| 위치 | 홈의 카드 슬롯 |
| 조건 | **오늘 체크인을 마쳐야 등장** ("보상 only" 원칙) |
| 동작 | 탭 → 카드 1장 무작위 공개 |
| 빈도 | 하루 1장. 다음 날 새 카드 |
| 미수행 시 | 슬롯 자체가 안 보임 |

- 카드는 **이름 + 키워드 + 하띠의 한 줄**로 구성.
- ⚠️ **덱 전체가 긍정/중립이다. 나쁜 운세는 없다.**
  감정 웰니스 앱에서 이미 가라앉은 사람에게 "오늘은 운이 나쁘다"를 주는 것은 해가 된다.
  전통 타로의 부정 카드(탑·죽음 등)는 덱에서 제외하거나 재해석해 넣는다.
- 체크인 완료 후 홈에 돌아올 **이유**를 만드는 장치이기도 하다.

## 5. 하띠 대사 규칙 (모든 화면 공통)

| | 내용 |
|---|---|
| **역할** | 공감. 오직 공감만 |
| **말투** | 따뜻한 반말. 곁에 앉은 친구 |
| **길이** | 2문장 이내 |
| **금지** | "힘내"·"파이팅"·"괜찮아질 거야" / 조언·해결책·지시 / 평가·판단 / 이모지·해시태그 |
| **금지** | 감정 라벨을 그대로 읊기 (X: "너는 피로를 느끼고 있구나" → O: "오늘 하루, 참 많이 버텼구나") |

> 이게 무드 트래커(분류만 함)·AI 챗봇(교정하려 함)과의 차별점이자 제품의 정체성.

## 6. 데이터 정의 (유저 관점)

| 저장되는 것 | 어디에 | 유저에게 보이는가 |
|---|---|---|
| 감정 원문 | `checkin_log.raw_text` | 히스토리에서 조회 가능 |
| 감정 분석 결과 | `checkin_log.emotion/intensity/context_keyword` | 감정 칩·태그로 표시 |
| 하띠 대사·확언 | `checkin_log.empathy/affirmation` | 응답 화면 |
| 친밀도·스트릭·단계 | `hatti_state` | 홈 상단 |
| 하띠 일기 | `checkin_log.diary` | 체크인 후 생성(비동기) |

- 감정 원문은 **민감 개인정보**. 본인만 조회 가능(RLS).
- 유저 요청 시 전체 삭제 가능해야 함.

## 7. MVP에서 하지 않는 것 (명시적 Out of scope)

| 미포함 | 이유 |
|--------|------|
| **유저의 감정 직접 선택/교정** | **의도적 배제.** 감정을 버튼으로 고르게 하는 순간 Daylio가 된다. "AI가 자연어에서 자동 추출, 라벨링 부담 제거"가 핵심 차별점 |
| 푸시 알림 | Day 2 내 불가. 리텐션 핵심이지만 로드맵 이관 |
| 감정 통계 대시보드 | "분석"은 하띠의 성격이 아님. 우선순위 낮음 |
| 소셜·공유 | 감정 데이터 공유는 신중해야 함 |
| 회원가입·로그인 화면 | 익명 로그인으로 대체 (앱 삭제 시 하띠 소실 — MVP 한계) |
| 감정 수정·삭제 | 히스토리 조회만 |

## 부록. 데모 시나리오 (발표용)

| # | 입력 | 보여줄 것 |
|---|------|-----------|
| 1 | "오늘 회의만 세 개였어. 진짜 너무 지친다" | 풀 루프 — 정밀 라벨링(피로/강도4/회의 과부하) → 공감 → 확언 카드 |
| 2 | (2회차) "또 야근이야…" | **하띠의 기억** — "지난번에도 회의가 너를…" + 성장/스트릭 토스트 |
| 3 | "다 사라지고 싶어" | **위기 분기** — 공감 대신 109 연결. 안전 설계 시연 |

> **시드 필수:** 시나리오 2의 "기억"은 과거 기록이 있어야 발동. 데모 계정에 미리 넣어둘 것.
> **경계선 테스트:** "우울해", "너무 힘들어"가 위기로 오판되지 않는지 발표 전 확인.
